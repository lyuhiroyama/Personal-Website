  • CSS
    • (def)
      • Cascading Style Sheets
      • Components:
        .someClass {
  display: flex;
}

.someClass -> Selector
display: flex; -> Declaration
display: -> Property
flex; -> Value
    • Origin
      • There exists 3 types of origins where the property values applied (defined by the Cascade algorithm) originates from:
        • User-agent stylesheets
          • (mdn) Default CSS rules applied by web browsers.
            • *User-agent : Browsers
        • Author stylesheets
          • (mdn) Styles written by web developers.
          • Defined by linking imported stylesheets, <style> blocks, or inline styles via the style attribute.
        • User stylesheets
          • (def) Custom stylesheet ccreated by a user to override/modify default styles of a webpage.
          • (e.g.) Right-click → Inspect
            • Deepending on the user-agent (broswer), user styles can also be added via browser extensions.
    • Cascade
      • (mdn) An algorithm that defines how user agents (browsers) combine property values originating from different origins (stylesheets):
        • User-agent stylesheet
        • Author stylesheet
        • User stylesheet
      • Cascade → Algorithm to do with stylesheets and which of them takes precedence.
      • Specificity → Algorithm to do with property values and which of them takes precedence.
    • Specificity
      • (def)
        • (mdn) The algorithm used by browsers to determine the CSS declaration that is the most relevant to an element, which in turn, determines the property value to apply to the element.
          • Specificity → Algorithm to do with property values and which of them takes precedence.
          • Cascade → Algorithm to do with stylesheets and which of them takes precedence.
      • ‘Order of importance’
        • !important > inline styles > specificity > rule order
        • Specificity seems to consider each concepts in the above order.
        • *term ‘order of importance’ might be made up by chagGPT.
      • ‘Rules’ defined by Specificity: (: inline styles/rule order)
        • (1) Inline styles: Styles applied to HTML elements directly takes precedence over internal/external styles.
        • (2) ID selectors > Class selectors > Type selectors
          • #someID takes precedence over .className which takes precedence over h1.
          • Of the three above, type selectors have the lowest specificity value.
        • (3) Combinators such as +/~/>, grouping/chaining selectors, and

          * (uniersal selector) have no specificity value.
          • Hence h1 takes precedence over *.
        • (4) Multiple clases take precedence over a single class.
          • Code:
            <div class="main">
  <div class="list subsection">Red text</div>
</div>

.subsection {
  color: blue;
}

.main .list {
  color: red;
}

<!-- color red would be applied here. -->
        • (5) ID + Class takes precedence over a single Class.
          • Code:
            <div class="main">
  <div class="list" id="subsection">Red text on yellow background</div>
</div>

#subsection {
  background-color: yellow;
  color: blue;
}

.main #subsection {
 color: red;
}

/* 
color red is applied. 
*/

/* 
background-color yellow is applied, since
no there are no conflicting declaration for it. 
*/
          • If there exists a conflict for a specific property between two selectors,
            but not for other properties in either selectors,
            the non-conflicted properties will be applied regardless of specificity.
            (See ‘Code:’ above.)
        • (6) Rule order: “Tie-breaker of tie-breakers”. Whichever rule/property to be specified last defined takes precedence.
          • One of the last fasctors considered when determining precedence of styles.

  • Adding CSS to HTML
    • External CSS
      • (def)
        • Creating of a separate file for CSS and linking it within the opening & closing of <head> tag, with the <link> element.
        • Useful when a website has many pages that share similar styles.
        • “styles.css” or “style.css” seems to be most commonly used for css file names.
      • Code
        <!-- index.html -->

<head>
		<link rel="stylesheet" href="styles.css">
		<!-- or -->
		<link rel="stylesheet" href="../directory/styles.css">
</head>
    • Internal/Embedded CSS
      • (def)
        • Adding of the CSS within <style> tags within the opening & closing <head> tags.
        • Can be useful for adding unique styles to a single page of a website.
      • Code
        <!-- index.html -->

<head>
	<style>
		p {
		  color: red;
		}
	</style>
</head>
    • Inline CSS
      • (def)
        • Adding of styles directly to HTML elements.
          • Overrides styles implemented using external/internal CSS.
        • Useful for adding unique style for a single element, though not recommended.
          • Can get messy pretty quickly. Bloats the HTML file.
      • Code
        <!-- index.html -->

<body>
	<div sytle="color: white; background-color: black;">Text</div>
</body>

  • Selectors
    • (def)
      • Patterns used to select/target HTML elements to apply styles.
      • <div class=”value1 value2 value3”></div>
        • The code above shows a class attribute with three different values. That’s a thing you can do in CSS.
    • Universal Selector (*)
      • * { /*styles here*/ }
        • (mdn) The universal selector (*) is a selector that matches elements of any type.
    • Type selectors
      • (mdn) Selectors that matches elements by node name (e.g. div, h2, etc.)
      • (e.g.)
        div {
		/* styles here */
}
    • Class selectors
      • (mdn) Selectors that matches elements based on the contents of their class attribute.
      • (e.g.)
        .className {
		/* styles here */
}
    • ID selectors
      • (mdn)
      • (e.g.)
        #id_name {
		/* styles here */
}
    • Grouping selectors (h1, h2, h3)
      • (def) Selectors where two or more selectors are grouped to cut down on repetition. Each separated by commas.
      • (e.g.)
        h1, h2, h3 {
		/* styles here */
}

----------------------------------------------------------------------

.className1, .className2, .className3 {
		/* styles here */
}
    • Chaining selectors (div.className)
      • (def) Selectors where two or more selectors are chained together to target elements that simultaneously match all specified selectors.
        • *Chaining more than one type selectors won’t work.
      • (e.g.)
        <div class="class_name"></div>

div.class_name {
	/* Combied type selector 'div' with class selector '.class_name' */
}

----------------------------------------------------------------------

<h3 id="id_name"></h3>

h3#id_name {
	/* Combied type selector 'h3' with ID selector '.#id_name' */
}

  • Combinators
    • Descendant combinators (.class1 .class2{})
      • (mdn) Combinator that combines two or more selectors separated by a single space each. Matches elements when the later most selector have an ancestor (be it a parent, parent’s parent, etc.) matching the preceding selector.
      • (e.g.)
         .class1 .class2 {
	/* styles here */
}

----------------------------------------------------------------------

<div class="class1">
		<div class="class2"></div>
</div>
    • *other combinators exists also.

  • Properties
    • color
      • (def)
        • (mdn) Property which defines the text color of an element.
        • Colors can be specified using many different keywords & notations.
      • Values:
        • keywords
          • Like blue/red/transparent/etc.
        • RGB Hexadecimals
          • #000000 : Black
          • #FFFFFF : White
          • #FF0000 : Red
          • #00FF00 : Green
          • #0000FF : Blue
        • rgb()
          • (mdn) Functional notation expressing a color according to its red, green, blue components.
          • rgb(0, 0, 0) : Black
            • rgb(0%, 0%, 0%) : Black (Percentage means “Percent of 255”)
            • rgb(0, 0, 0, 50%) : Half-transparent black. Fourth value is opacity.
          • rgb(255, 255, 255) : White
            • rgb(255%, 255%, 255%) : White
          • rgb(255, 0, 0) : Red
          • rgb(0, 255, 0) : Green
          • rgb(0, 0, 255) : Blue
        • Other types of values
          • hsl()
          • hwb()
          • lab()
          • lch()
    • background-color
      • (def)
        • (mdn) Property that sets the background color of an element.
        • Values used seems to be same as the color property.
    • font-family
      • (def)
        • (mdn) Property that specifies a prioritized list of one or more

          font family names or generic family names for the specified element.
          Requires quotations when there are whitespaces.
          • If a browser cant find / doesn’t support the first font in a list, it uses the next one, then the nexxt one, and so on until it finds a supported/valid font.
          • Default values seems to vary by browsers / user settings.
        • *Font family names :
          • (def) Names of a font family
          • (e.g.) ”Times New Roman”/”Arial”/”Verdana”
        • *Generic family names :
          • (def) Fonts that can be used as a last alternative. Should be last item in the list of font names.
          • (e.g.) serif/sans-serif/monospace/cursive
      • (e.g) font-family: “Times New Roman”, sans-serif;
    • font-size
      • (def) Sets the size of the font.
      • font-size: 22px;
      • font-size: 3em;
      • font-size: 25%;
        • <percentage> : Percentage value relative to the parent element’s font size.
    • font-weight
      • (def)
        • (mdn) Sets the weight (boldness) of the font. Weights available depends on the

          font-family that is currently set.
        • Uses numeric or keyword values.
      • font-weight: 100;
        • (def) Numerical values generally ranges between 100-900, lightest to boldest.
      • font-weight: normal;
        • (mdn) Normal font weight. Same as the value 400.
      • font-weight: bold;
        • (mdn) Bold font weight. Same as the value 700.
      • font-weight: lighter;
        • (mdn) One relative font weight lighter than the parent element.
      • font-weight: bolder;
        • (mdn) One relative font weight heavier than the parent element.
    • text-align
      • (def)
        • (mdn) Property which sets the horizontal alignment of the
          inline-level content inside a block element /
          table-cell box.
        • *Table-cell box : Refers to the rectangular box that corresponds to a cell in a table. Where contents of <td> / <th> are displayed.
      • text-align: left;
        • Inline contents gets aligned to the left edge.
      • text-align: right;
        • Inline contents gets aligned to the right edge.
      • text-align: center;
        • Inline contents gets horizontally centered.
      • text-align: start;
        • Same as left if direction is set to left-to-right.
      • text-align: end;
        • Same as right if direction is set to left-to-right.
      • text-align: justify;
        • Inline contents gets justified.
          • *Justify : Adjust so that the print fills a space evenly.
    • height
      • (def)
        • (mdn) Specifies the height of an element.
          • By default, defines the height of the content area.
          • box-sizing: border-box; would have height/width define values for the border area instead.
        • Good practice to always assign <img> elements height/width values.
          • ‘Reserves’ a space on the page for the image.
          • When values aren’t assgined, if an image takes longer to load than the rest of the page contents, the image won’t take any place on the page at first, but will suddenly cause a drastic shift of the other contents once the image does load.
      • height: 100px;
      • height: 100em;
      • height: 100vh;
      • height: 25%;
        • (def) Defines the height as a percentage of the containing block’s height.
      • height: auto;
        • (def) Browser calculates and selects a height for the specified element.
          • (w3schools) The element automatically adjusts its height to allow its content to be displayed correctly.
        • Can also be used to have things like images/videos adjusts the size of itself without losing proportions, by setting one value to a desired height/weight and the other the value auto.
          • (e.g.) img { height: auto; width: 200px; }
    • width
      • (def) Same as height but for width. (see section under: ‘height’)
    • box-sizing
      • (def)
        • (mdn) Sets how the total height/width of an element is calculated.
      • box-sizing: content-box;
        • (mdn) Gives you the default box-sizing behavior (width/height values gets only applied to the element’s content box/area.)
        • padding/margin/border values are not accounted for.
      • box-sizing: border-box;
        • (mdn) Tells the browswer to account for any padding/border values you specify for the element’s width/height.
          • Note that it doesn’t account for the margin area.
        • Gets the width/height values to define the height/weight of the border area (as opposed to the content area).
    • border
      • (def)
        • (mdn) Shorthand property that sets an element’s border.
        • Shorthand for:
          • <border-width, border-style, border-color>
          • All values above are optional.
      • Properties shorthand for
        • border-width:
          • thin
          • medium
          • thick
          • 2px
          • 2px 1em 0 2cm; (top, right, bottom, left)
        • border-style:
          • none (no borders)
          • hidden (shows no borders but still occupies space)
          • solid (solid line)
          • groove (carved look)
          • ridge (opposite of groove)
          • inset (embedded look)
          • onset (embossed look. opposite of inset)
      • (e.g.) <border-width, border-style, border-color>
        • solid;
        • 2em none blue;
        • thick dashed #FF0000
    • padding
      • (def)
        • (mdn) Shorthand property that sets the padding area on all 4 sides of an element at once.
        • Shorthand for:
          • <padding-top, padding-right, padding-bottom, padding-left>
      • (e.g.) <padding-top, padding-right, padding-bottom, padding-left>
        • 2em (all 4 sides)
        • 25% (percentage relative to the inline size of the containing block)
          • Inline size
            • Size defining the horizontal or vertical size of an element’s block, depending on its writing mode.
            • writing mode by default is set to the value horizontal-tb.
        • 10px 20px 30px (top, right&left, bottom)
        • 10px 20px 30px 40px (top, right, left, bottom)
    • margin
      • (def)
        • (mdn) Shorthand property that sets the margin area on all 4 sides of an element at once.
        • Shorthand for:
          • <margin-top, margin-right, margin-bottom, margin-left>
      • (e.g.) <margin-top, margin-right, margin-bottom, margin-left>
        • auto
          • (def)
            • (mdn) Browser selects a suitable margin to use.
            • Commonly used to center a block level element horizontally.
            • auto takes up the the available horizontal space in the element’s container equally, thus centering the element.
          • Prerequisites for it to work:
            • Must be a block-level element.
            • Must have a specified width.
          • Code:
            body {
	width: 30em;
}

p {
	width: auto; margin-left: auto; margin-right: auto;
}
        • 2em (all 4 sides)
        • 25% (percentage relative to the inline size of the containing block)
          • Inline size
            • Size defining the horizontal or vertical size of an element’s block, depending on its writing mode.
            • writing mode by default is set to the value horizontal-tb.
        • 10px 20px 30px (top, right&left, bottom)
        • 10px 20px 30px 40px (top, right, left, bottom)
      • margin: auto;
        • (def) Browser selects a suitable margin to use. Commonly used to center an element horizontally.
        • To horizontally center an element using margin: auto: (source: stack)
          • Element must have a defined width that is not auto.
          • Element must be block-level.
          • Element must not float.
          • Element must not have a fixed/absolute position.
          • Exceptions/Caveats: (stackoverflow link)
    • display
      • (def)
        • (w3sch) Specifies the display behavior of an element.
        • Some elements defaults to display: block, and some to display: inline.
      • display: none;
        • Turns off the display of the element and all of its descendant elements.
        • It’s space it would have occupied disappears also. (Unlike visibility: hidden;)
      • display: inline;
        • (mdn) The element generates one (or more) inline boxes that

          do not generate line breaks before and after themselves.
      • display: block;
        • (mdn) The element generates a block box, generating line breaks before and after the element.
      • display: inline-block;
        • (w3sc) Displays an element as an inline-level block container. Element is formatted as an inline element, but you can apply height/width values
          (unlike
          display: inline;).
          • So an inline-element that you can add height/width values to.
        • (odin) Useful tool to know about, but in practice you’ll likely end up reaching for flex box if you’re trying to line up boxes.
      • display: flex;
        • (mdn) Element behaves like a block-level element and lays out its content according to the flexbox model.
      • display: grid;
        • Element behaves like a block-level element and lays out its content according to the grid model.
    • position
      • (def)
        • (mdn) Sets how an element is positioned in a document. (Sometimes) Used together with the top/right/bottom/left properties, which determine the final location of positioned elements (depends on what value is assigned to position).
      • position: static; (default)
        • *default value for position.
        • static: Element is positioned according to the normal flow of the document.
          → Properties
          top/right/bottom/left/z-index have no effect.
      • postion: relative;
        • relative: Element is positioned according to the normal flow of the document, then offset relative to itself based on the values of top/right/bottom/left.
        • Code
          .someClass {
	position: relative;
	top: 10px;
}

Code above moves the said element 10px below. It's as you are
telling the element "I'll add 10px of space on top of you."
      • position: absolute;
        • absolute: removes the element from the normal document flow, and no space is created for the element in the page layout. Element is positioned relative to its closest positioned ancestor, or to the initial containing block.
          • Optionally used together with top/right/bottom/left property.
          • *positioned ancestor:
            • (def) Closest ancestor to the said element with a set position value

              other than the static value.
            • Reference from which top/right/bottom/left properties are calculated, when said element is is set to position: absolute;.
      • position: fixed;
        • fixed: removes the element from the normal document flow, and no space is created for the element in the page layout. Element is positioned relative to its initial containing block, that being, the viewport in the case of visual media.
          • “Fixed position elements are positioned based on the entire HTML element.”
        • Useful for making an element remain in the same place on the viewport as you scroll.
      • position: sticky;
        • sticky: Positioned according to the normal flow of the document, then offset relative to its nearest scrolling ancestor & containing block (including table-related elements, i.e., <table> <tr> <td> <th>), based on the values of the

          top/right/bottom/left property.
          • *scrolling ancestor
            • (def) Element containing another element with its overflow property set to anything but visible (So hidden/scroll/auto/overlay, etc.).
            • However, position: sticky still might work with overflow set to visible, due to browser implementations apparently. If you want it to work reliably, then I’m guessing you assign overflow to anything aside from visible.
          • *When no scrolling ancestor / containing block exists for an element with sticky value, the <body> tag will be used as its containing block.
        • (e.g.) top: 10px;

          → means “offset 10px from the top of the containing block”.

    • overflow
      • (def)
        • Sets the desired behavior when content does not fit in the parent element box.

  • Flexbox
    • (def)
      • (mdn) A one-dimensional layout method for arranging items in rows or columns. Items ‘flex’, as in expand to fill additional space, or shrink to fit into smaller spaces.
        • *Flex items themselves can simultaneously be a flex container and have a set of flex items themselves.
      • Allows you to style the flexibility/sizing of different elements in the flex container, from the flex container selector, without having to style the flex items.
    • How-to
      • Establish a flex container and a set of flex items.
      • Make the flex container flexible by declaring display: flex;
      • Declare flex container / flex item properties of your choice:
        • *See under “Properties” section for details.
        • justify-content
        • align-items
        • flex-direction
        • flex-wrap
        • flex-flow
        • align-content
        • gap
        • flex
      • Apply desired values to declared properties.
    • Properties
      • Flex container properties:
        • flex-direction
          • (def) Defines the primary axis along which flex items are placed inside a flex container, establishing either a main axis (horizontal) or cross axis (vertical), as normal or reversed.
          • flex-direction: row; (default)
            • (def) Arranges flex items horizontally along the main axis,
              from left to right.
          • flex-direction: row-reverse;
            • (def) Arranges flex items horizontally along the main axis,
              from right to left.
          • flex-direction: column;
            • (def) Arranges flex items vertically along the cross axis,
              from top to bottom.
          • flex-direction: column-reverse;
            • (def) Arranges flex items vertically along the cross axis,
              from bottom to top.
        • justify-content
          • (def) Defines how the browser distributes space between & around flex items along the main axis of the flex container.
          • justify-content: flex-start;
            • (mdn) Items are packed flush to each other, toward the edge of the alignment container depending on the flex-container’s starting edge of the main axis.
              • *left end of the main axis by default. Determined by the value of flex-direction.
              • flex-start & flex-end takes the value of flex-direction in to account.
                The values
                start & end does not.
          • justify-content: flex-end;
            • (def) Reverse of flex-start
          • justify-content: center;
            • (mdn) Items are packed flush to each other toward the ceter of the alignment container along the main axis.
          • justify-content: space-between;
            • (mdn) Items are evenly distributed within the alignment container along the main axis.
              • The first item is flush with the starting edge of the main axis, and the

                last item is flush with the ending egge of the main axis.
          • justify-content: space-around;
            • (mdn) Items are evenly distributed within the alignment container along the main axis.
              • Empty space before the first, and after the last item equates to half of the space between each pair of adjacent items.

          • justify-content: space-evenly;
            • (mdn) Items are evenly distributed within the alignment container along the main axis.
              • Spacing between
                each pair of adjacent items,
                the starting edge of the
                main axis & the first item,
                and the ending edge of the
                main axis & the last item
                are set to equal exactly the same.
        • align-items
          • (def) In flexbox, it controls the alignment of flex items on the cross axis.
          • align-items: flex-start;
            • (mdn) Used in flex layout only. Aligns flex items flush against the flex container’s starting edge of the cross axis.
          • align-items: flex-end;
            • (mdn) Used in flex layout only. Aligns flex items flush against the flex container’s ending edge of the cross axis.
          • align-items: center;
            • (mdn) Flex items’ margin boxes are centered within the line on the cross axis.
          • align-items: stretch;
            • (mdn) If the items are smaller than the aligment container, auto-sized itesm will be equally enlarged to fill the container, while respecting the items’ width & height limits.
          • align-items: baseline;
            • (def) Takes the first element in each flex item, and lines things up by the bottom of that first element. (link)
        • flex-wrap
          • (def)
            • (mdn) Sets whether flex items are forced onto one line or can wrap onto multiple lines. If wrapping is allowed, it sets the direction that lines are stacked.
          • flex-wrap: nowrap; (default)
            • (mdn) Flex items are laid out in a single line which may cause the flex container to overflow.
          • flex-wrap: wrap;
            • (mdn) Flex items break into multiple lines.
          • flex-wrap: wrap-reverse;
            • (mdn) Behaves the same as wrap but cross(axis)-start and cross-end are permuted (reversed).
        • flex-flow (shorthand prop.)
          • (mdn) Shorthand property for flex-direction & flex-wrap. (link)
        • align-content
          • (stack) align-content determines the spacing between lines, whereas align-items determines how the items as a whole are aligned within the container. When there is only one line within a flex container,

            align-content has no effect.
            • (mdn) Sets the distribution of space between and around content items along a flexbox's cross-axis (or a grid's block axis).
        • gap
          • (def)
            • (mdn) Shorthand property which sets the gaps (aka gutters) between rows and columns.
              • Assigning one value sets both <row-gap> & <column-gap>.
              • Assigning two values sets the first to <row-gap>, and the second to <column-gap>.
          • gap: 20px;
            • Adds 20px between each flex items.
      • Flex item properties:
        • flex (shorthand prop.)
          • (def)
            • Shorthand property for the following 3 properties:
              • flex-grow (default value: 0)
                • (def) Determines the proportion of a space a flex item takes up within a flex container along the main axis, allowing it to expand relative to other flex items based on its assigned value.
                  • *Default value seems to be 0 to signify that by default, the property the values is assigned to won’t grow beyond their initial size unless specified otherwise.
              • flex-shrink (default value: 1)
                • (def) Determines the proportion of a space a flex item can shrink within a flex container along the main axis, allowing it to shrink relative to other flex items based on its assigned value.
                  • *Default value seems to be 1 rather than 0 because ‘0’ assigned to flex-shrink seems to particularly signify/instruct the flex item to not shrink at all.
              • flex-basis (default value: 0 or auto)
                • (mdn) Sets the initial main (axis?) size of a flex item. Sets the size of the content box, unless otherwise declared with box-sizing.
                  • Sets the size of the width if flex-direction is set to row/row-reverse, and height if flex-direction is set to column/column-reverse.
                • *When omitted from the flex shorthand, its specified value is 0%.
                  In the case of
                  flex-basis (not short-hand), value defaults to auto.

  • Glossary
    • Inline-level content
      • (mdn) Content that participates in inline-layout.
        • Are displayed in a line. They do not force the text after them to a new line.
      • (e.g.) <a> <strong> <b> <i> <em> <span>
    • Block-level content (: block element)
      • (mdn) Content that participates in block-layout.
        • Each block-level element starts a new line on the page. Takes up as much horizontal space as possible.
        • ‘Block element’ seems to be used interchangeably.
      • (e.g.) <div> <p> <h1> <h2> <ol> <ul> <dl>
    • Containing block
      • (mdn) Most often is the content area of an element’s nearest block-level ancestor,
        but this is not always the case.
        • Exceptions: look it up
          → (One of them: table-related elements like
          <table> <tr> <td> <th>)
    • Content area
      • (mdn) Area containing the ‘real’ content of the element, such as text/image/video player.
      • Excludes any padding/margin/border.
    • Padding area
      • (mdn) Area extending the content area to include the element’s padding.
    • Border area
      • (mdn) Area extending the padding area to include the element’s borders.
    • Margin area
      • (mdn) Area extending the border area to include the element’s margins.
    • Normal flow (Normal document flow)
      • (mdn) aka ‘Flow layout’ is the way that block & inline elements are displayed on a page before any changes are made to their layout.
    • Viewport
      • (mdn) Area of the window in which web content can be seen.
