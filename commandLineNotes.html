Command Line (1)

  • CLI
    • (def)
      • *Command-line Interface
      • (def) Text-based UI used to run programs.
      • Different CLI's:
        • Mac → Terminal
        • Windows → Command Prompt
        • Linux: Linux Bash Shell

  • Unix Shell
    • (def)
      • *The ‘Shell’
      • Both a CLI & a scripting language.
        A ‘Shell’ that provides an interface between the user and the OS (’kernel’).
        • *scripting language (wiki) : Programming language used to manipulate/customize/automate the facilities of an existing system.
      • Different types of Unix Shells:
        • sh - Bourne shell
        • bash - Bourne again shell (Was written as part of the GNU project)
        • zsh - Z shell
    • *Shell Script
      • (def) File that contains commands/instructions written in a shell scripting language.
      • File extension formats: .sh

  • Shell Commands
    • Components of a Shell Comand:
      • $ ls -F /

        → $ : Prompt (Indicates where you can type commands)
        →
        ls : Command
        →
        -F : Option
        →
        / : Argument
    • Shortcuts and stuff
      • tab key : Used for auto completion.
      • . : To refer to the current working directory.
      • .. : To refer to the parent directory (of the current working directory).
      • cd : This command alone will get you back to your home directory.
      • ~ : To refer to the home directory.
      • ctrl + c: Terminates the command.
      • ctrl + a: Move to the begenning of a line.
      • ctrl + c: Move to the end of a line.
      • ↑/↓ : To navigate between command history to access previously executed commands.
        • !! : Retrieves the very previous command executed (alternative to ↑)
        • !$ : Retrieves the last word of the last command. (Useful for long filenames)
      • ctrl + r:
        • (def)
          • Called the ‘reverse-i-search’ (’i’ stands for ‘incremental’.)
            → *It ‘incrementally’ searches previous commands as you type.
        • To use:
          • Press ctrl + r to enter ‘reverse-i-search’ mode.
          • Type in a keyword you want to search through history with.
            → A result will be displayed.
          • Press ctrl + r to choose between other search results.
          • Either press enter to execute,
            or use
            ← and → keys to make edits before executing.
    • Options
      • (def) Additional settings that can be provided in addition to a command.
        • Adds changes to the command.
        • aka: ‘Flags’, ‘Switches’
        • Short Options: Begins with a single dash (-)
        • Long Options: Begins with two dashes (--)
      • (e.g.) ls -F (’-F’ is the option)
      • To use multiple options within a single command (both -r & -t options):
        →
        ls -rt
    • Arguments
      • (def) Simple parameters given on the system’s commandline.
        • Tells the command what to opeate on.
      • (e.g.) ls -F Desktop/
        • Desktop is an argument.
          → (The trailing
          / seems to be a convention to note that Desktop is a directory)
        • -F is an option.
        • This command lists the files in the Desktop directory, each classified with trailing symbols.
    • Operators (>, >>, | )
      • >
        • (def) Redirection operator for:
          • Redirect the output of a command to a new file.
          • If argument file already exists: Overwrites the file entered if file already exists.
      • >>
        • (def) Redirection operator for:
          • Appending the output of a command to a file entered as argument.
          • If argument file already exists: appending the output to the already existing file,
      • |
        • (def)
          • Redirection operator used to run multiple commands in a single line.
            • Called a ‘Pipe’
          • The output of the first command gets used as the input for the second command.
        • sort -n someFile.txt | head -n 1
          • Sorts the contents of someFile.txt numerically, then outputs the very first line of the newly sorted content.
    • ls
      • (def)
        • Lists files in the current directory.
        • Short for ‘listing’.
      • ls -F
        • (def) The -F ‘option’ tells ls to classsify the output to indicate whether each output is a directory, link, or an executable. (With a trailing symbol)
        • (e.g.)
          • someDirectory/

            → Indicates a directory.
          • someLink@

            → Indicates a link
          • someExecutable*

            → Indicates an executable
          • *Output with no symbol indicates files in the current working directory.

      • ls -a
        • (def) Lists all directories and files, including hidden ones, in the current working directory.
        • Stands for ‘show all’.
        • . : Refers to the current working directory.
        • .. : Refers to the parent directory.
      • ls -l
        • (def) Lists files in the long format, which provides the following information for each file: permissions/# of links/owner/group/size/timstamp/name
      • man ls
        • (def) Turns your terminal into a page with a description of the ls commands and its options.
        • To quit: Press ‘q’
      • ls --help
        • (def) Displays information on how to use the command or program.
        • *Doesn’t seem to work on your Terminal.

    • cd

        • Short for ‘change directory’.
        • Akin to double-clicking in a GUI.
        • This command actually changes the shell’s current working directory,
          rather than
          changing the directory as the command might have you think.
      • cd
        • (def) Command which returns you to the home directory.
        • Same as the cd ~ command.
      • cd -
        • (def) Changes the current working directory to its previous directory.
      • cd .
        • (def) Changes the current working directory to the current directory, essentially changed nothing.
        • . references the current working directory.
      • cd ..
        • (def) Changes the current working directory to its parent directory.
      • cd ~
        • (def) Command which changes your current working directory to the current user’s home directory.
        • cd ~/Desktop/anotherFolder : Abbreviates what would have been /Users/lyuhiroyama with ~.
      • man cd
        • (def) Turns your terminal into a page with a description of the cd commands and its options.
    • clear
      • (def) Clears the screen.
    • mkdir
      • (def)
        • Command used to make directories with.
      • mkdir folder1 folder2
        • Creates two directories with one command.
      • mkdir -p folder1/folder2/
        • Creates a directory hierarchy where folder2 is a subdirectory of folder1.
        • -p : Causes mkdir to to create intermediate subdirectories as required.
    • nano
      • (def)
        • Refers to the command-line text editor ‘GNU nano’. Exists within the Terminal app.
      • nano
        • (def) Opens the nano text editorwith an empty buffer.
          • (i.e.) opening nano without a specified file to edit.
          • *Buffer : Temporary memory area in which data is stored while it is being processed/transferred.

    • rm
      • (def)
        • Removes/Deletes files (forever).
        • Removing a directory (and not a file) requires the -r option.
      • rm textFile.txt
        • (def) Removes the text file.
      • rm -r someDirectory
        • Deletes the someDirectory directory.
      • rm -i textFile.txt
        • Removes textFile.txt, but prompts the user before deletion for confirmation.
    • mv
      • (def)
        • Command for moving files & directories. Also for renaming files.
        • ‘moving’
      • mv textFile.txt ~/Desktop/
        • (def) Moves the textFile.txt in the current working directory to the ~/Desktop/ directory.
      • mv textFile.txt newName.txt
        • (def) Changes the name of the file (in the current working directory) textFile to newName.
      • mv Desktop/textFile.txt Downloads/newName.txt
        • (def) Changes the name of the textFile.txt file in the Desktop directory,
          to
          newName.txt and moves the file to the Downloads directory.
    • cp
      • (def)
        • Command for copying files/directories.
      • cp textFile.txt Desktop/duplicatedFile.txt (Copying files)
        • Copies the textFile.txt file and pastes it into the Desktop directory with the name duplicatedFile.txt.
      • cp -r folder/ backupFolder/ (Copying directories?)
        • Copies the entire folder directory and pastes it in the backupFolder directory.
        • -r : idk. something to do with recursion.
      • cp textFile.txt textFile2.txt someDirectory (Copying multiple files)
        • As shown. When copying multiple file names to a directory, the destination directory must be the last argument.
        • When copying multiple files, shell expects the last argument to be a directory (and not a file).
    • *
      • (def)
        • A wildcard which represents one or more characters.
        • Wildcards are useful for working with multiple files at once. (link)
    • ?
      • (def)
        • A wildcard which represents exactly one characters.
        • Wildcards are useful for working with multiple files at once. (link)
    • wc
      • (def)
        • Command which counts the number of lines, words & characters in a file.
        • ‘word count’
      • wc someFile.txt
      • wc *.txt
        • Gets the word count information for all files in the current working directory with the .txt extension. Utilizes the * wildcard.
      • wc -l someFile.txt
        • Only displays the line count.
      • wc -w someFile.txt
        • Only displays the word count.
      • wc -m someFile.txt
        • Only displays the character count.
      • wc -l *.pdb > textFile.txt
        • Gets the line count for every file with a .pdb suffix, then redirects the output to a separate textFile. Could be a new file or a pre-existing one (which you would then overwrite).
        • > : Redirection operator
    • cat
      • (def)
        • Reads data from the file and gives its content as output.
        • Accepts multiple files as argument. Will concatenate then output.
        • ‘concatenate’
      • cat textFile.pdb
        • Outputs contents of the textFile.pdb file.
      • cat textFile.pdb textFile2.pdb

        • Command for displaying a screenful of a file, then stops.
          (As opposed to dumping the whole content of a file like
          cat does).
        • *Screenful : A unit of content that fits within the current size of the Terminal window.
        • Controls:
          • spacebar : Move one screenful. (Might be something else in Terminal idk)

        • Displays lines of text/string which as passed as arguments.

    • sort
      • (def)
        • Command for sorting the contents of a file.
        • This command does not change the file. It simply displays the sorted result in the output.
        • Sorts in an alphabetical order by default, without an option.
      • sort textFile.txt
        • Sorts the contents in the file in an alphebatical order.
      • sort -n textFile.txt
        • Sorts the content in the file in a numerical order.
      • sort textFile.txt > newFile.txt
        • Sorts the contents of textFile.txt, then places the alphabetically sorted contents in a new file newFile.txt (or overrides it).
    • head
      • (def)
        • Outputs a specified number of lines/bytes of the start of a specified file(s).
      • head -n 1 textFile.txt
        • (def) Outputs the first line of the textFile.txt file.
        • -n : Sets the unit used for the amount of content to output to the nuber of lines.
        • 1 : Sets the number of selected unit to display (in this case # of lines) to 1.
      • head -n 20 textFile.txt
        • Outputs the first 20 lines of the textFile.txt file.
    • tail
      • (def)
        • Outputs a specified number of lines/bytes of the end of a specified file(s).
      • tail -n 1 textFile.txt
    • cut
      • (def)
        • Command for cutting out sections from each line of files, then displaying in the output.
        • Uses tab spaces as separaters as default. Can implement a delimeter of your choice using -d option.
        • Requires one of one of the following options to be used to specify what unit to use to select which part of the file to cut:
          • -b, -c, -f (bytes/chatacter/fields)
          • *Field : Refers to columns.
      • cut -d , -f 2 someCSV.csv
        • -d (parameter: delim )
          • (man) Use delim as the field delimeter character instaed of the tab character.
            • *tab character : Tab space.
          • , is the argument in this case. Will be used as the delimeter.
        • -f (parameter: list)
          • (man) The list specifies fields, separated in the input by the field delimeter character specified with the -d option. Output fields are separated by a single occurence of the field delimeter character.
          • So you enter a number to indicate what field/column of the someCSV.csv file you want to cut, which in this case is 2.
        • someCSV.csv
          • The file you want to perform the cutting on.

  • For-loops
    • (def)
      • The $ symbol within the for-loop denotes that the word attached to it is a variable.
      • The > symbol that appears after entering the first line of the for-in loop denotes that you haven’t finished typing a complete command yet.
      • The ; (semicolon) can be used to separate two commands written on a single line.
      • Loop Variable:
        • The loop variable (’filename’ in the first code example under ‘Code:’) can be referred to by either $filename or ${filename} (They are one and the same).
        • This is useful for not only clarity but to alter the loop variables
          (e.g.
          ${filename^}, ${filename[0]}) (Capitalization & Subscripting, respectively)

    • Code:
      /* A simple for-in loop through multiple files */

$ for filename in file1.dat file2.dat file3.dat
> do
>	  echo $filename
>   head -n 2 $filename | tail -n 1
> done


------------------------------------------------------------------------
/* To make copies of multiple files using for-in loop */

$ for filename in *.dat
> do
>   cp $filename original-$filename
> done

------------------------------------------------------------------------
/* To see what each iteration in the for-in loop above would output: */

$ for filename in *.dat
> do
>   echo cp $filename original-$filename
> done

------------------------------------------------------------------------
/* 
To execute a shell script on multiple .txt files in a directory
using for-in loop:
*/

$ for data_file in *.dat
> do
> echo $data_file
> bash shellScript.sh $data_file stats-$data_file
> done

------------------------------------------------------------------------
/* Nested For-in loop */

$ for species in cubane.pdb ethane.pdb methane.pdb
> do
>     for temperature in 25 30 37 40
>     do
>         mkdir $species-$temperature
>     done
> done
