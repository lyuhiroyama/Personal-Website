  • DOM Manipulation
    • DOM (def) (: Node)
      • (w3c) An API for HTML & XML documents.
        • “A programming API for documents.”
        • Closely resembles the structure of the documents it models.
      • Tree-like data representation of nodes (objects) that comprise the structure & content of a document.
        • *Document : Refers to a single, self-contained unit of content on the web. Essentially a webpage.
        • *Node : “Simply an HTML element. Every HTML element is a ‘node’.”
          → *In a broader sense, “
          node” refers to a basic unit of information/data used to build data structures.
    • Origin
      • Originated as a specification to allow JS scripts & Java programs to be portable among web browsers.
    • How it works
      • In the DOM, documents have a logical structure like a tree (Technically more like a forest).
        • Structural model : Technical term for the tree-like representation of a document.
      • When a browser loads an HTML page, it converts it to the Document Object Model.
        The browser’s produced HTML DOM, constructs a tree that consists of all your HTML elements as objects.
        • Scripting languages like JS uses the DOM to access the documents and its elements. JS does not alter your HTML, but instead the DOM.

    • DOM Node Interface (: and its properties/methods)

  • DOM Glossary
    • NodeList
      • (mdn) Collections of nodes (node objects). Not an array.
        • What the property Node.childNode / method document.querySelectorAll() returns.
      • 2 types of NodeList’s:
        • 1) Live NodeLists
          • (def) NodeList where changes in the DOM automatically updates the NodeList.
          • (e.g.) Return value of Node.childNode.
        • 2) Static NodeLists
          • (def) NodeList where changes in the DOM do not get automatically update the NodeList.
          • (e.g.) Return value of Node.querySelectorAll()
    • Node methods
      • (def) Methods defined within the Node interface.
    • Document methods
      • (def) Methods defined within the Document interface.
      • Called like this: document.methodHere().
      • *Document : Refers to the Document interface.
      • *document : Refers to the current HTML document. Instance of Document.

  • DOM related methods
    • querySelector(selectors)
      • (mdn) Document method that returns the first element (node object) within the document that matches the specified selector(s).
        • Returns null when no matches are found.
      • selector : String containing one or more selectors to match. Must be a valid CSS selector string.
    • querySelectorAll(selectors)
      • (mdn) Document method that returns a static NodeList representing a list of the document’s elements that match the specified selector(s).
      • selectors : String containing one or more selectors to match. Must be a valid CSS selector string.
    • createElement()
      • (def)
        • (mdn) Document method that creates an HTML element with the specified tagName.
        • Does not put the newly created element into the DOM. It simply creates it in memory (ram).
        • You can add it to the DOM using appendChild() or insertBefore().
          You can remove it from the DOM using
          removeChild().
      • createElement(tagName)
        • tagName : String that specifies the type of element to be created.
          • Initializes the nodeName of the created element with tagName.
          • Not case-sensitive.
        • (e.g.) createElement(”div”)
      • createElement(tagName, options )
    • appendChild()
      • (def)
        • (mdn) Node method that adds a node (node object) to the end of the list of children of a specified parent node. Returns the newly appended node.
      • parentNode.appendChild(childNode)
        • parentNode : Parent node to which the newly created child node will be appended
        • childNode : Child node to be appended to the parent node.
    • insertBefore()
      • (def)
        • (mdn) Node method that inserts a node before a reference node as a child of a parent node (which would be the node this method calls on).
      • parentNode.insertBefore(newNode, referenceNode)
        • newNode : The node (under the parent node) to be inserted.
        • referenceNode : The node (under the parent node) you reference to place
          the
          newNode before(preceding) it.
    • removeChild()
      • (def)
        • (mdn) Node method that removes a child node from the DOM, and returns the removed node.
      • parentNode.removeChild(child)
        • child : Node that is the child node you want to remove from the DOM.

  • Variables
    • (def)
      • Variable names are case sensitive.
      • camelCase is the convention.
      • First character must not be a number.
      • Can only contain letters/digits, or the symbols $ & _. (No hyphens!)
      • Reserved words: let/class/return/function/etc.
    • var
      • (def)
        • (mdn) Declares function-scoped or globally-scoped variables. Re-declarable.
        • (stack) Difference between var & let: Variables declared by var are scoped to the immediate function body (hence the function scope), whille let variables are scoped to the immediate block.
          • var declared within a function → only available within the function (function-scoped).
          • Does not apply to other code blocks like if, for, while, etc.
        • *Re-declaring & Re-assigning:
          • var lavar = “neva lost” has both actions.
            →
            var : Declaration
            →
            = : Assignment
      • Snake Case is the convention.
        • myVariable = 100;
    • let
      • (def)
        • (mdn) Declares block-scoped local variables. Re-assignable. Unre-declarable.
          • *Block-scoped : (def) The scope created with a pair of curly braces ( { } ).
        • (stack) Difference between var & let: Variables declared by var are scoped to the immediate function body (hence the function scope), whille let variables are scoped to the immediate block.
      • Snake Case is the convention.
        • myVariable = 100;
    • const
      • (def)
        • (mdn) Declares block-scoped local variables. Unre-assignable. Unre-declarable.
          • *constant objects can have properties added/updated/removed.

      • Uppercase Camel Case is the convention.
        • MY_CONSTANT = 100;

    • Anonymous Function
      • (mdn) Function without a name. Often used when a function expects another function as a parameter.
        • Anonymous Function seems to refer to the ‘function’ part of the Function Expression syntax, that being, the right side of the ‘=’ assignment operator.
      • Code:
        /* Anonymous Function used as a parameter: */

button.addEventListener("click", function () {
		alert("Button clicked!");
})

/* Anonymous Function used as part of a Function Expression: */

let exclaim = function() {
		console.log("Neva Lost!");

    • Call stack
      • (mdn) Mechanism for an interpreter to keep track of its place in a script that calls multiple functions.
        • When a script calls a function: Interpreter adds it to the call stack, and starts carrying out the function.
        • Any additional functions called by that function are added to the call stack further up, and run when their calls are reached.
        • If the call stack takes up more space than it was given, a “stack overflow” error is thrown.
      • (def)
        • A data structure - a ‘stack’ to be precise.
        • Operates on the LIFO principle (Last in, first out).

  • For loops
    • for(let = 1; i < 10; i++) (Standard for-loop)
      • (def) Statement creating a loop consisting of three optional expressions, followed by a statement to be executed in the loop.
        • Second argument: When condition becomes falsy, the loop is terminated.
          • *break statement can also be used to terminate loop.
        • Third argument gets executed after each iteration.
      • Standard for-loop:
        for (let i = 0; i < 3; i++) {
	console.log(i)
}

// output: 0 1 2
      • Single-line for-loop:
        for (let i = 0; i < 3; i++) console.log(i);

// output: 0 1 2
    • for (const varName of iterableObj) (for…of loop)
      • (mdn) Statement which executes a loop that operates on a sequence of values sourced from iterable objects such as Array/String/Map/Set/NodeList, etc.
        • Iterates over element as opposed to for…in loops. for…in loops seem to iterate over each indices/properties.
      • Code:
        const ballBros = ["Lonzo", "LiAngelo", "LaMelo"];

for (const bigBaller of ballBros) {
	console.log(bigBaller);
}

/* 
output:
Lonzo
LiAngelo
LaMelo
*/
    • for (const key in object) (for…if loop)
      • (mdn) Statement which iterates over all enumerable string properties of an object.
        • Iterates over each indices/properties as opposed to for…of loops. for…of loops seem to iterate over each element.

    • while (i < 3) (while-loop)
      • (def)
        • (mdn) Statement which creates a loop that executes a specified statement as long as the test condition evaluates to true. Condition is evaluated before executing the statement.
          • Condition: Seems to be required. Expression evaluated before each pass through the loop. Then again, it seems to also execute the code as well. Having (++i < 5) as the condtion, the condition both evaluates and executes itself upon every iteration, seemingly proven by ‘i’ being incremented upon every loop.
      • Standard while-loop:
        let i = 0;

while (i < 3) {
	alert(i);
	i++;
}
      • Single-line while-loop:
        /* Example 1 */
-----------------------------------------------------------------
let i = 3;
while (i) console.log(--i);
//output: 2 1 0
//Condition becomes falsy when i = 0


/* In long form: */
let i = 3;
while (i) {
	console.log(--i); 
}
//or
while (i) {
	console.log(i = i - 1);
}
//output: 2 1 0
//note: prefix decrement operator returns value after decrementing.


/* Example 2 */
-----------------------------------------------------------------

let i = 0;
while (++i < 5) alert(i);
//output: 1 2 3 4

/* In long form: */
let i = 0;
while (++i < 5) {
	alert(i);
}
//output: 1 2 3 4
    • do {} while (i < 3) (do…while loop)
      • (mdn) Like while-loop but condition is evaluated after executing the statement. Thus results in the specified statement executing at least once.
        • Standard while-loop seems to be preferred unless you need it to execute a least once.
      • Code:
        let i = 0;

do {
	console.log(i);
	i++;
} while(i < 3);

//output:
// 0
// 1
// 2
    • break statement
      • (mdn) Terminates the current loop/switch statement and transfers program control to the statement following the terminated statement.
      • Code:
        let sum = 0;

while (true) {
	
	let value = +prompt("Enter a number:", '');
	
	if (!value) break; //single-line if statement.	
	
	sum += value;
}
    • continue statement
      • (def)
        • (js.info) “Lighter version” of break.
        • (mdn) Terminates execution of the statements in the current iteration, and continues execution of the loop with the next iteration.
      • Code:
        for (let i = 0; i < 10; i++) {

	if (i % 2 == 0) continue;
	//if i is an even number, current iteration is terminated,
	//and is moved on to the next iteration.

	alert(`Your odd numbers: ${i}`);
}
    • loopLabel: for () {} (Labeled statement)
      • (js.info) Identifier with a colon used as a label for loop/switch statements. Allows you to jump to this label using either break or continue statement that is nested within the labeled statement.
        • Commonly used for escaping a nested looop.
      • Code:
        loopLabel: for (let i = 0; i < 3; i++) {
	
	for (let j = 0; j < 3; j++) {
		let input = prompt(`Value at cell (${i},${j}):`, "");

		if (!input) break loopLabel;
		// if user submits empty string or cancels,
		// control flows out of the loopLabel loop,
		// thus exiting BOTH loops.

	}
}
