HTML (1)

  • Non-tags / Glossary
    • Tag
      • (def) Specific markers such as <p> and </p> that indicates the beginning/end of an element.
    • Element
      • (def) What consists of an opening tag, optional content, and a closing tag.
    • Attribute
      • (mdn) What provides additional behavior/metadata to an element.
    • <!DOCTYPE html>
      • (def)
        • (w3c) Declaration entered on top of all HTML documents to specify a specific version of HTML/XHTML.
          • *Not a tag/element! Effect of it being a declaration rather than a creation of an element, perhaps.
            → Thus in won’t be rendered as a node object in the DOM.
        • Makes sure the document will be parsed the same way by different browsers.
      • <! DOCTYPE html>
        • (w3c) Declaration of the use of the HTML versions HTML5 & beyond.
        • Case-insensitive.
      • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
        • Doctype declaration for HTML4.01.
        • Won’t use.

  • Elements
    • <html>
      • (def) Element representing the root of an HTML document. Aka root element.
        All other elements must be descendants of this element.
      • lang = “en” : Attribute specifying english as the language for a webpage. Optional.
        This helps screen reading technologies to determine the proper language to announce.

    • <head>
      • (def) Element containing metadata (machine-readable information) about the document.
      • (e.g.) title/scripts/style-sheets
    • <link>
      • (def)
        • (mdn) Element which specifies relationships between the current document and an external resource.
        • Used for linking stylesheets/favicons/etc.
      • <link href=”main.css” rel=”stylesheet”>
        • (def) Provides path to stylesheet inside the href attribute.
        • rel : Value denoting how the item beink linked is related to the containing document.
    • <meta>
      • (def) Element representing metadata unable to be represented by other HTML meta-related elements.
      • <meta charset=”utf-8”>
        • charset : Giving the character encoding in which the document is encoded.
        • *utf-8 : 8-32 bits used to express UNICODE each character.
      • <meta name="viewport" content="width=device-width, initial-scale=1.0">
        • name : Sets the <meta> element to privide document-level metadata applying to the whole page.
          • ”viewport” : Tells the browser that this meta tag is defining settings for the viewport.
          • *viewport : Area in computer graphics being currently viewed. Everything on the screen that is visible.
        • content : Attribute use together with either the name or http-equiv attribute.
          • "width=device-width, : Sets the width of viewport to match the scree nwidth of the device.
          • initial-scale=1.0" : Sets the initial zoome level when the page is loaded. 1.0 means that the web page is shown at 100% zoom.
    • <title>
      • (def) Element defining the document’s title that is shown in a
        browser’s title bar / page tab.
        • Without it, it defaults to the filename (index.html).
      • (e.g.) <title>Facebook</title>
    • <body>
      • (def) Element representing the content of the HTML document.
        There can only be on of these within a document.
    • <a>
      • (def) Anchor element. Used together with href attribute, creates a hyperlink to anything that a URL can address (web pages/files/email address/etc.).
      • (e.g.) <a href=”https://bigballerbrandinc.com” target=”_blank”

        rel=”noopener noreferrer”>Some link</a>
      • href
        • (mdn) URL that the hyperlink points to. Not restricted to HTTP-based URLs.
          Can use any URL scheme
          (email/phone number/ document, text, media fragments).
        • ”https://bigballerbrandinc.com”
          • An absolute link leading to the BBB page.
          • *Absolute link : Links to pages on other websites.
            Typically looks like this:
            protocol://domain/path
        • "./directory/about.html" :
          • A relative link leading to another page on the same site. Comes in the form of a file path. Assumes the domain name will be the same.
      • target
        • (def) Specifies where to display the linked URL.
        • _self : (default) Displays linked URL on the currrent browsing context.
        • _blank : Usually a new tab. Users can configure browsers to open a new window instead.
        • _parent : Displays on the parent browsing context of the current one. If no parent, behaves as _self. Good practice to pair it with rel=”noopener noreferrer” .
        • _top : Displays on the topmost browsing context (the “highest” context that’s an ancestor of the current one). If no ancestors, behaves as _self.
      • rel
        • (def) The relationship of the linked URL as space-separated link types.
        • noopener : Used with target=”_blank”.
          • (def)
            • Newly opened window/tab has access to the window object by default via window.opener property. Which can be a security risk because the new page can potentially redirect the orignal page to a malicious URL.
              This is known as ‘tabnabbing’.
            • noopener prevents the new page from accessing the window.opener property, effectively mitigating the security risk.
            • This is especially relevant to websites that you do not trust completely.
        • noreferrer : Used with target=”_blank”.
          • (def)
            • Like noopener, it prevents the new page from accessing the window.opener property.
            • It also prevents the browser from sending the referer (original page) HTTP header when the new page is requested. This keeps the nw page from knowing where the traffic came from.
            • Useful when you don’t want the linked page to know that you’re sending traffic to them.
    • <img>
      • (mdn) Element that embeds an image into the document.
        • Self-closing tag.
        • *HTML5 doesn’t require slashes for self-closing tags. XHTM/HTML4 does.
        • (e.g.) <img src=”../media/lavar.jpg” alt=”Lavar Headshot”

          height=”300” width=”300”>
        • src : Required. Contains the path to the image you want to embed.
        • alt : Mandatory apparently. Describes images for screen readers & accessibility purposes.
        • height/width : Optional. Sets dimensions in pixels.
    • <div>
      • “Simply an empty container.”
        • Allows us to divide the page into different blocks and apply styling to those blocks.
      • Block-level element by default.
    • <span>
      • (mdn) Inline container which does not inherently represent anything.
        • Used to group text content and inline HTML elements for styling.
      • Inline-level element by default.
